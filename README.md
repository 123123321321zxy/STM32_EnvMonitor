# 基于 STM32+FreeRTOS 的环境监测系统

## 1. 项目简介
本项目实现了一个基于 STM32F103 和 FreeRTOS 的环境监测系统。系统实时采集温湿度（DHT22），通过 OLED 显示，并支持按键控制（启动/停止采集）和高温 LED 报警。采用多任务架构，使用队列进行任务间通信。

**主要功能**：
- 实时采集温湿度，每 2 秒刷新一次
- OLED 显示当前温湿度
- 短按按键启动采集，长按按键停止采集
- 温度超过 35℃ 时 LED 闪烁报警
- 系统上电后采集任务默认挂起，需按键启动

## 2. 硬件连接
| 外设 | 引脚 | 说明 |
|------|------|------|
| DHT22 | PA0 | 单总线数据线（需外接 4.7kΩ 上拉,DHT22模块集成上拉电阻） |
| OLED (I2C) | PB6 (SCL), PB7 (SDA) | 0.96 寸，地址 0x78 |
| 按键 | PB0 | 低电平有效，双边沿触发 |
| LED | PC13| 低电平点亮，高温闪烁 |
| USART1 | PA9 (TX), PA10 (RX) | 调试打印，波特率 115200 |

## 3. 软件架构

### RTOS 任务划分
| 任务名称 | 功能 | 优先级 | 周期/触发 |
|----------|------|--------|----------|
| SensorTask | 读取 DHT22，校验后发送到队列 | 3 | 2 秒（vTaskDelay） |
| OledTask | 阻塞接收队列数据，更新 OLED | 2 | 事件驱动 |
| KeyTask | 处理按键事件（短按/长按） | 2 | 事件驱动（队列+中断） |
| LedTask | 温度超限时 LED 闪烁 | 1 | 事件驱动（任务通知） |

### 任务间通信
- 传感器数据通过OledQueue发送，OledTask读取。
- 按键中断将事件（时间戳+电平）发送到 KeyQueue，KeyTask 负责长短按事件判断。

### 中断设计
- 按键采用双边沿触发，中断中做硬件消抖（间隔 <20ms 丢弃），并将 时间戳和按下或松开事件入队。
- 中断服务函数遵循快进快出原则。
- 设定长按超时时间为1s，松开事件超时不入队

## 4. 关键实现细节
- **DHT22 驱动**：使用微秒级延时（硬件定时器），按单总线协议读取 40 位数据，校验通过后更新。
- **按键消抖**：中断中记录上一次触发时间，小于 20ms 则丢弃，保证任务中接收的均为有效边沿。
- **长按/短按判断**：KeyTask 状态机根据按下与松开的时间差区分长按（≥500ms）和短按。
- **高温报警**：LedTask 读取温度，≥35℃ 时以  LED闪烁。

## 5. 使用说明
1. 上电后 OLED 显示“Ready”，采集任务挂起。
2. 短按按键 → 启动采集，OLED 开始刷新温湿度。
3. 长按按键（≥500ms）→ 停止采集，OLED 显示“Stopped”。
4. 温度超过 35℃ 时 LED 闪烁；温度恢复正常后 LED 熄灭。

## 6. 技术栈
- **MCU**：STM32F103C8T6
- **RTOS**：FreeRTOS
- **IDE**：STM32CubeIDE
- **调试工具**：逻辑分析仪、串口助手、ST-Link

## 7. 遇到的问题与解决
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| DHT22 读不到数据 | 起始信号拉低时间不足 | 增加至 2ms，用逻辑分析仪验证 |
| 偶尔读到 0 或校验失败 | 时序临界或干扰 | 失败时保持上次有效数据 |
| 按键长按后短按无反应 | 中断状态未正确复位 | 改用中断触发事件，任务中实现状态机 |
